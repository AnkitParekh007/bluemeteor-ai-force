General

  • Agent outputs require human review before merge, send, or production use.
  • Agents may be wrong, incomplete, or confidently incorrect.
  • Do not paste secrets, tokens, customer PII, or regulated data you would not put in an internal ticket.
  • Do not rely on generated code or SQL without review and testing.

Browser / Testo

  • Browser testing is intended for configured local or staging targets.
  • Production testing may be blocked by policy.
  • Some flows require manual login capture for authenticated sessions.
  • Generated Playwright specs should be reviewed before CI adoption.

Connectors

  • Some connectors may fall back to mock mode when credentials are missing.
  • Pilot posture is read-first; do not assume write actions to Jira/Confluence/etc.
  • No automated pull request creation from agents in this phase.

MCP

  • MCP tools are oriented read-only first; write-like tools remain gated or blocked.

Dato

  • SQL is generated for review only — no live execution against databases from the agent.

DevOpsy

  • Release checklists and pipeline review only — no deployment execution from the agent.

Security

  • Pilot is for internal, controlled usage.
  • Report permission issues, suspected data leaks, or policy bypasses immediately to your admin contact.

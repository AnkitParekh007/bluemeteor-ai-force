How to report issues

Open a ticket with your internal IT or platform team using the template below. For urgent security concerns, follow your company’s security incident channel.

Include this information

  • Agent name and slug
  • Prompt or task summary (redact secrets)
  • Run ID from the workspace or activity panel when available
  • Trace ID from the session header (copy button) when available
  • Screenshot if UI-related
  • Expected vs actual behavior

Finding trace ID

In the agent workspace, when a run is active or completed, the header shows a trace chip when the backend provides a trace ID — use copy to attach to your ticket.

Feedback vs incidents

Use Pilot → Feedback for product feedback (quality, usefulness). Use the issue template for break/fix and access problems.

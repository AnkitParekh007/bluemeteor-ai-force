# Pilot user guide

## How to log in

Use your company credentials on the app login page. If access fails, contact your admin — roles and `agents.*` permissions control what you can open.

## How to choose an agent

Open **Agents** or follow **Pilot → Onboarding** for the recommended agent for your role.

## How to start a session

Inside an agent workspace, create a new session from the sidebar. Pick **Ask**, **Plan**, or **Act** according to what your administrator enabled.

## How to use sample prompts

Click **Samples** in the session header (lightbulb). A prompt inserts into the composer — review and edit before **Send**. Longer examples live under **Pilot → Agent guides**.

## How to review artifacts

Use the right-hand **Artifacts** (and **Browser** / **Tests** when enabled). Treat everything as draft until you have reviewed it like a peer’s PR.

## How to use the browser / test panel

Prefer local or staging URLs. Do not drive production flows unless your organization explicitly allows it. Capture auth only when instructed by admins.

## How to submit feedback

Use **Feedback** in the workspace header (pre-fills agent and run/trace when available) or **Pilot → Feedback**. Be specific: what worked, what failed, rating, optional time saved.

## Do / don’t

- Do: redact secrets, use approvals, cite run/trace IDs in support tickets.
- Don’t: paste credentials, auto-send customer replies, merge unreviewed code.
